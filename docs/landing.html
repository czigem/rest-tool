rest-tool

Create and maintain a well-defined REST API specification, which provides a solid basis for implementing and mocking the RESTful interface between the parties that communicates with each other through it.

Get Started

Rapid development

Create a working REST API project from scratch in less than 15 minutes

Using the tool the frontent and backend developers can work fully independently from each other. They can assure that their implementation is fully comply to the REST API specification. The API specification is functioning as a DMZ and contract between them.

The 'rest-tool' relies on the HTTP protocol specification, and NOT on language specific implementations. You can keep your communicating agents loosely coupled, and fully technology agnostic.

Instant mocking (static/dynamic)

Use the mock Node.js server for testing, and simulating of the REST API functionality.

Use the mock server's to provide your web UI frontend source as static content during the development, without the need for deployment.

Use the mock server's proxy feature to test your web UI frontend diretly against 3rd party application servers (JBoss, Tomcat, Apache, etc.) without deployment.

Automated testing and validation

Generate/Update test cases which can be used to test either the mock server and the real test/production servers to verify that the server is working according to the API.

Define your own templates to generate test cases and server side implementations which best fit to your needs. (the server side templating is currently under development).

Instant documentation

Generate/Update the HTML format documentation of the REST API

Specify the resources (uniquely identified by their URL patterns) that can be provided/retrieved through the RESTful services.

Specify the expected working of HTTP methods in relation to each resource.

Put additional information beside the resource definitions (mock data, validation schemas, etc.)
